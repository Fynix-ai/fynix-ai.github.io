Rules for AI

Home Docs Home Features Overview

Use Rules for AI to add custom rules for Fynix Copilot. These rules provide additional context and guide the behavior of Fynix Copilot for each chat or result it produces.

Steps to Add Rules

  1. Go to the Settings Tab in the Assistant Chat View and click on Advanced Settings.
  2. Under Rules for AI, add your custom rule.
  3. Once added, the rule is automatically applied, and you’ll see a “Settings Updated Successfully” pop-up on the bottom-right of VS Code.

Use Cases

1. Directory Structure Awareness

When updating a repository's structure, copy and paste the directory structure into "Rules for AI". This allows Fynix Copilot to provide accurate responses based on the project's layout. 


Example Directory:
project-root/
├── index.html
├── css/
│   ├── styles.css
│   ├── reset.css
├── js/
│   ├── main.js
│   ├── utils.js
├── images/
│   ├── logo.png
│   ├── banner.jpg
├── assets/
│   ├── fonts/
│   ├── icons/
├── README.md
├── .gitignore
└── package.json

Example: Typing delete banner in the Fynix Terminal will generate: 

$ rm project-root/images/banner.jpg && echo 'banner.jpg has been successfully deleted.'

2. Enforcing Project-Specific Coding Standards

Scenario: Ensure consistent coding practices for multi-developer projects.

Rule: “All variable names must follow snake_case, and functions must include docstrings with parameters and return types.”

Workflow: Use /fix or /doc commands to refactor code or generate documentation adhering to these rules.

3. Tailored Responses for Domain-Specific Applications

Scenario: Fintech applications require compliance with security standards.

Rule: “Ensure no sensitive data (e.g., SSN, credit card numbers) is logged in plain text. Use encryption for sensitive fields.”

Workflow: Use /fix to enforce security practices, and add the same rule in AI Review to flag any violations.

4. Enforcing Best Practices for New Hires

Scenario: Ensure junior developers follow best practices for React or Django.

Rule: “All React components must be functional components, and PropTypes should be defined for each component.”

Workflow: Use /new to scaffold best-practice components and /fix to align legacy code.

5. Automating Environment-Specific Adjustments

Scenario: A project runs in multiple environments (e.g., development, staging, production).

Rule: “Use mock APIs for development and staging environments. Use production APIs only in production builds.”

Workflow: Use /complete to generate code with environment-specific configurations and include this rule in AI Review to prevent exposing production credentials.
